Some Technical Description

dhttpr uses inetd to receive and respond to requests using the standard input and output streams. This allows the network/socket/communication heavy lifting to be done by an external application, and greatly simplifies the operation of the software I have to write.

The original couple of iterations of the software were called directly by inetd, but as the feature set increased, the amount of code loaded to serve each request grew alarmingly. The current version runs as a daemon, with inetd invoking a lightweight process to shuffle each request to the server process and subsequent response back.
